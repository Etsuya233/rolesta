# Chat 左栏与会话编辑器设计

## 背景

当前 Chat 左栏已经提供聊天列表、当前聊天、创建、编辑和删除能力，但界面存在明显的尺寸与节奏问题：Tab 与工具区之间留有多余空白，搜索图标和新增按钮的视觉尺寸不协调，筛选控件在窄栏中分配宽度不稳定，会话条目的信息层级也与角色卡、预设等资产模块脱节。

创建和编辑会话目前使用居中 Dialog。会话配置需要同时选择角色卡、Persona、预设和模型连接，普通下拉列表难以承载头像、摘要、筛选、分页和多种展示方式。大尺寸表单 Dialog 也切断了左栏本身的页面导航关系。

本次改造统一 Chat 左栏的页面模型和视觉密度，并为不同资产提供各自适配的选择 Dialog。

## 目标

- 修正聊天列表的间距、控件尺寸、宽度分配和信息层级。
- 将新增和编辑会话从居中表单 Dialog 迁移到左栏内的保活页面栈。
- 保留被编辑页覆盖的聊天列表或当前聊天页面状态。
- 为角色卡、预设和模型连接提供专属选择 Dialog。
- 让角色选择器支持列表和图标网格展示。
- 保留现有会话表单的默认资产、标题跟随角色和预设关联模型规则。
- 同步整理当前聊天详情页的间距和资产摘要样式，不增加新业务功能。

## 非目标

- 不修改 Chat、角色卡、预设或模型连接的后端契约。
- 不改变会话领域规则、资产访问规则或默认资产优先级。
- 不增加新的资产类型、批量选择或多选能力。
- 不重做删除确认流程。
- 不新增或执行自动化测试；最终视觉和交互由用户手工验证。

## 方案选择

采用 Chat 模块内的完整页面栈，并在选择器中共享基础交互、保留各资产的专属展示。

保留现有组件结构并局部替换 Dialog 的方案改动较小，但会继续把主页状态、表单状态和弹层状态集中在 `ChatSidebar`。建立全局配置驱动的通用资产选择框架复用范围更大，但角色卡、预设和模型连接的视觉模型差异明显，当前范围不需要承担复杂配置层。

选定方案与 Preset、世界书已经使用的 `KeepAliveStack` 一致，同时把复用限制在稳定的选择行为上。

## 页面架构

Chat 左栏根组件使用 `KeepAliveStack` 管理以下页面：

- 会话主页：栈底页面，承载“聊天列表 / 当前聊天”两个 Tab。
- 新增会话页：从主页推入，覆盖整个左栏。
- 编辑会话页：从主页推入，覆盖整个左栏，并记录进入来源。

新增或编辑页覆盖顶部 Tab。主页保持挂载，因此列表的搜索、筛选、排序、分页和滚动位置，以及当前选中的 Tab 都会保留。资产选择 Dialog 打开时，编辑页继续挂载，Dialog 关闭后表单内容保持不变。

用户从新增或编辑页返回时，页面从栈中移除，草稿随页面一同销毁。表单存在未保存改动时，返回操作先打开放弃更改确认 Dialog。确认放弃后弹出页面；取消确认时继续留在编辑页。本次不保留已经退出页面的草稿。

新增成功后将新会话设为活动会话，返回主页并切换到“当前聊天”。编辑成功后返回进入编辑页之前的 Tab：从列表进入则回到列表，从当前聊天进入则回到当前聊天。删除会话继续使用现有确认 Dialog。

活动会话被删除或详情明确返回 404 时，清除活动会话并切回聊天列表。普通详情请求错误保留当前页面和活动会话，允许用户重试。

## 聊天列表布局

左栏支持现有约 `224–448px` 的可调整宽度，并与角色卡、预设模块使用同一套基础节奏：

- 页面主要区域使用 `16px` 水平内边距。
- 工具区使用 `12px` 纵向内边距和 `12px` 行间距。
- 同一行控件使用 `8px` 间距。
- Tab 与工具区直接衔接，只保留表达区域边界的分隔线。
- 搜索框与相邻图标按钮使用一致的标准控件高度。
- 搜索图标位于输入框内部，使用标准图标尺寸。

第一行工具栏放搜索框和新增按钮。第二行放角色筛选器和紧凑的排序 Popover。角色筛选器占据剩余宽度；排序按钮使用固定正方形尺寸，避免两个宽 Select 在窄栏中竞争空间。

会话条目采用预设列表的全宽分隔行结构，并加入角色卡列表的 `40px` 头像：

- 条目左右内边距为 `16px`，上下内边距为 `12px`。
- 标题位于第一行，单行截断。
- 角色名和更新时间位于第二行，均使用次要文本样式。
- 缺少角色时使用明确的缺失状态图标和文本。
- 更多菜单按钮固定为 `32px`，不会压缩正文区域。
- 当前会话通过稳定的背景色高亮，不改变条目尺寸、边框或内边距。

底部分页使用适合左栏的紧凑布局。窄宽度下展示每页数量、上一页、当前页和下一页；空间足够时可以显示首页和末页。分页控件不得产生水平滚动或挤压相邻元素。

## 当前聊天布局

当前聊天页与聊天列表共用顶部、正文和分隔线节奏。标题、会话 ID、编辑和删除操作组成紧凑顶部区域。正文中的角色卡、Persona、预设和模型连接改为一致的资产摘要行：

- 角色卡和 Persona 展示头像、名称及缺失状态。
- 预设展示名称。
- 模型连接展示名称、提供商类型和默认模型名称。
- 未选择的可选资产使用统一的空状态文案。
- 创建时间和更新时间与资产配置分组展示。

本次只调整信息组织和视觉样式，当前聊天页不增加新的命令或配置字段。

## 新增与编辑页

新增和编辑共用会话表单内容，页面结构分为三个稳定区域：

- 顶部返回栏：返回按钮和“新增会话”或“编辑会话”标题。
- 中间滚动区：标题输入框和四个资产字段。
- 底部提交区：固定的全宽“创建”或“保存”按钮。

页面不提供单独的取消按钮，返回按钮承担退出操作。提交期间禁用重复提交。编辑表单没有实际变化时禁用保存按钮。

标题继续在编辑页内直接输入。创建表单在用户首次手动编辑标题前，选择对话角色会将角色名称写入标题；用户手动编辑标题后，后续更换角色不再覆盖标题。

角色卡、Persona、预设和模型连接使用可点击的资产摘要字段。字段展示当前选中资产的名称和适合该资产的最小摘要，点击后打开对应的选择 Dialog。对话角色为必填字段，只允许替换；Persona、预设和模型连接允许清除。

现有默认资产加载、手动操作锁定、本人预设关联模型自动带入和候选置顶规则保持不变。默认资产加载失败时在表单中保留重试入口，重试不得覆盖用户已经手动操作的字段。

## 资产选择 Dialog

选择器共享稳定的弹层结构与选择行为，各资产拥有专属内容组件。共享能力包括：

- 响应式 Dialog 外壳。
- 搜索输入。
- 可选的筛选、排序和展示方式工具。
- 分页。
- 暂存选择、清除、取消和确认。
- 加载、空结果、错误与重试状态。

桌面端 Dialog 使用约 `720–880px` 的中大型宽度，并限制最大高度。窄屏下接近全屏：顶部固定标题与关闭按钮，中间内容独立滚动，底部固定取消和确认操作。文本和操作不得被底部区域遮挡。

Dialog 顶部工具栏将搜索框放在主要宽度区域。筛选和排序使用搜索框右侧的图标按钮打开 Popover；角色选择器额外提供列表与图标网格切换。工具栏在窄屏下允许合理换行，但按钮尺寸和内容区域宽度保持稳定。

用户单击资产条目只更新 Dialog 内的暂存选择。搜索、筛选、排序、切换展示方式和翻页不会直接修改会话表单，也不会清除暂存选择。点击确认后才把选择写入表单；取消或关闭 Dialog 时丢弃本次暂存变化。

打开 Dialog 时以表单当前值作为初始暂存选择。当前资产不在已加载页面中时，仍通过表单已有摘要显示已选状态。可选资产的 Dialog 提供明确的清除选择操作；清除同样需要点击确认后生效。

### 角色卡与 Persona

对话角色和 Persona 使用同一个角色选择能力，通过用途决定标题、必填规则和默认 Persona 标记。候选范围包含用户拥有和当前公开可见的角色卡。

列表视图展示头像、名称、备注摘要、标签和可见性。图标网格以头像和名称作为主要识别信息，并保留必要的可见性状态。Persona 用途额外标记默认 Persona。展示方式作为本地 UI 偏好记住，后续重新打开角色选择器时沿用上次模式。

工具栏提供搜索、权限范围、排序和分页。权限范围包含全部、我的和公开。

### 预设

预设选择器使用高信息密度列表，展示名称、可见性、Token 数、条目数和使用次数，并标记默认预设。工具栏提供搜索、权限范围、排序和分页。

### 模型连接

模型连接选择器使用列表视图，展示连接名称、提供商类型和默认模型名称，并标记默认模型连接。候选只包含当前用户自己的连接。工具栏提供搜索、排序和分页，不显示权限范围筛选。

## 查询与数据边界

现有角色卡、预设和模型连接列表 API 已提供设计所需的摘要、搜索、排序、筛选和分页能力，本次预计不修改 OpenAPI 或后端模块。

选择器按实际页请求数据，不再一次请求最多 100 条候选。搜索输入使用短延迟触发查询；搜索、筛选、排序或每页数量变化时回到第一页。查询缓存沿用 TanStack Query，并使用包含资产类型和完整查询参数的稳定 query key。

编辑会话详情已经包含当前资产摘要；创建页的默认资产只有 ID。表单维护已选资产摘要，用户从选择器确认时一并记录候选摘要。遇到只有 ID 的默认资产或缓存恢复值时，通过现有资产详情接口解析摘要。解析期间展示加载状态，解析失败时展示资产不可用状态和重新选择命令，界面不回显原始 ID。

选择器只展示当前设计列出的摘要字段。模型连接列表响应中的 Base URL、凭据模式等字段不进入选择器界面。

## 错误处理

- 聊天列表加载失败时保留当前查询条件，并显示错误状态和重试命令。
- 选择器列表加载失败时保留暂存选择，并在内容区显示重试命令。
- 会话提交失败时保留完整草稿。
- 服务端返回资产不可用错误时，将错误显示在对应资产字段。
- 编辑会话加载返回 404 时退出编辑页、刷新列表并提示会话已不可用。
- 编辑会话普通网络错误时留在编辑页，提供重试和返回能力。
- 默认资产加载失败时保留用户已经填写或选择的内容。

## 可访问性与响应式要求

- 所有图标按钮提供可访问名称和工具提示。
- Dialog 使用明确标题，暂存选中项暴露选中状态。
- 列表和网格条目支持键盘聚焦与选择。
- 不活动的保活页面沿用 `inert` 与 `aria-hidden`，不接受焦点或指针事件。
- 左栏在最窄和最宽尺寸下均不得出现非预期水平滚动。
- 固定顶部和底部区域不得遮挡滚动内容。
- 文本截断只用于辅助信息；表单错误和关键状态必须完整可读。

## 验收方式

本次不新增或执行自动化测试。实现完成后执行 TypeScript 类型检查、Lint 和基础页面冒烟检查。最终视觉与交互由用户在实际工作区中手工验证，重点覆盖：

- 左栏最窄和最宽尺寸下的列表间距、控件宽度和分页布局。
- 新增与编辑页覆盖整个左栏，返回后主页状态保持。
- 退出新增或编辑页后草稿销毁，脏表单需要确认。
- 新增和编辑成功后的正确返回位置。
- 四类资产字段的选择、确认和清除规则。
- 角色列表与图标网格切换。
- 选择器搜索、筛选、排序 Popover 和分页。
- 桌面 Dialog 与 `390px` 窄屏下的高度、滚动和固定操作区。
- 加载失败、提交失败、资产失效和编辑目标 404 状态。
