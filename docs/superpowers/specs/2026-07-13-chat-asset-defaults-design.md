# 聊天资产默认值设计

## 背景

Rolesta 已支持角色卡、预设和模型连接管理，聊天功能尚未实现。用户需要提前为聊天上下文设置默认 Persona、默认预设和默认模型连接，使后续聊天创建流程能够读取并预选这些资产。

Persona 继续使用现有角色卡表达。本次仅记录作为用户身份使用的默认角色卡，不增加默认对话角色，也不设计 Persona 选择器、聊天创建页或聊天数据模型。

## 目标

- 每个用户可以分别设置一个默认 Persona 角色卡、一个默认预设和一个默认模型连接。
- 三个默认值相互独立，均允许为空，也允许单独取消。
- 默认角色卡和默认预设只允许使用当前用户拥有的资产；默认模型连接沿用现有本人所有规则。
- 在三类资产编辑页提供即时设置入口，并在资产列表中清楚展示当前默认项。
- 提供可供未来聊天创建流程直接读取的稳定接口。

## 领域边界

新增 `ChatPreferencesModule`，代码目录使用 `chat-preferences`。该模块负责当前用户的聊天偏好，资产默认值是模块下的第一个偏好资源。角色卡、预设和模型连接的领域对象及保存协议保持不变，默认状态也不进入任何导入或导出格式。

模块内部包含资产默认值的应用用例、持久化端口、Kysely 实现、HTTP 控制器、DTO 和错误映射。资产所有权校验由该模块协调，并在一次事务中完成校验与写入。三类现有资产模块无需承担默认值切换逻辑。

## 数据模型

新增 `asset_defaults` 表，每个用户最多对应一行：

| 列名 | 类型 | 约束 | 含义 |
| --- | --- | --- | --- |
| `user_id` | string | 主键，引用 `users.id`，删除用户时级联删除 | 偏好所属用户 |
| `persona_character_id` | string / null | 引用 `characters.id`，删除角色卡时设为 null | 默认 Persona 角色卡 |
| `preset_id` | string / null | 引用 `presets.id`，删除预设时设为 null | 默认预设 |
| `model_provider_id` | string / null | 引用 `model_provider_configs.id`，删除模型连接时设为 null | 默认模型连接 |

表记录在用户第一次修改默认值时通过 upsert 创建。尚无记录表示三个默认值均未配置。数据库外键负责在资产删除后清空对应列，读取流程不再额外探测资产是否存在。

数据库外键只能保证资源存在，无法单独保证资源与用户的所有权关系。应用层必须验证每个非空 ID 的 `owner_user_id` 与认证用户一致，并在全部验证通过后写入。

## HTTP API

资产默认值使用以下认证接口：

- `GET /chat-preferences/assets`
- `PATCH /chat-preferences/assets`

GET 返回当前用户的完整资产默认值：

```json
{
  "personaCharacterId": null,
  "presetId": null,
  "modelProviderId": null
}
```

PATCH 接受同名字段，每个字段均可省略或显式传入 null。省略表示保持原值，null 表示取消对应默认值，字符串 ID 表示设置新的默认资产。请求可以同时修改多个字段，空对象不构成有效更新。

```json
{
  "personaCharacterId": "character-1",
  "presetId": null
}
```

PATCH 成功后返回更新后的完整资产默认值。重复设置相同 ID 或重复清空同一字段均视为成功，使接口保持幂等。

## 所有权与原子性

角色卡和预设当前允许读取他人公开资产，但公开状态不授予默认值使用资格。任何非空 ID 都必须属于认证用户，模型连接同样按所有者查询。

一次 PATCH 涉及多个字段时，应用层在同一事务中完成全部非空 ID 的所有权校验和 upsert。任一字段验证失败时回滚整个操作，不保留部分修改。ID 不存在或不属于当前用户时，接口统一返回“资产不可用于默认值”的 404 错误，避免通过该接口探测他人的私有资产。

GET 在 `asset_defaults` 记录不存在时返回三个 null，明确表达用户尚未配置默认值。资产删除后的列清空由外键执行；删除用户时整行记录随用户删除。

## 前端交互

角色卡、预设和模型连接的编辑页顶栏增加默认值按钮。按钮只在资产已经保存且属于当前用户时显示；新增页、导入页以及他人公开资产页面不显示该操作。

未设为默认时按钮显示空心圆，当前默认时显示 Lucide `CircleCheck`。按钮通过 tooltip 和 `aria-label` 分别表达“设为默认”与“取消默认”，避免使用容易被理解为收藏的星标。点击后立即调用 PATCH，成功后更新统一的聊天资产偏好缓存，失败时沿用现有 Notification 展示错误。mutation 进行期间按钮禁用，防止连续提交造成状态闪动。

三类资产列表读取同一份默认值，并按 ID 给当前默认条目展示“默认”徽标。列表仅展示状态，不提供切换按钮。默认模型连接指向一条完整的现有模型连接配置，后续使用时一并获得该配置保存的 Provider、Base URL、凭据引用和默认模型名；本次不增加独立的默认模型或默认 API Key。

删除 mutation 成功后同时使资产列表查询和聊天资产偏好查询失效。若被删除项原本是默认值，重新读取后由数据库中的 null 移除对应徽标和选中状态。

## 前端数据流

Web 端新增聊天偏好 API 封装与统一查询，查询键固定用于资产默认值。角色卡、预设和模型连接管理器共享该查询结果，各自只比较对应 ID。默认值 mutation 成功后直接以服务端完整响应更新缓存，避免等待三类资产列表重新请求。

编辑页结合当前用户信息和资产详情中的 `ownerUserId` 决定是否显示按钮。模型连接均为本人资产，仍使用同一判断规则，保持三类页面行为一致。默认值状态不写入三类资产摘要响应，避免资产 API 与用户偏好产生额外耦合。

## 错误处理

- 未认证访问沿用现有认证错误。
- PATCH 空对象返回参数校验错误。
- 字段格式错误返回参数校验错误。
- 资产不存在或不属于当前用户时返回资产不可用的 404 错误。
- 多字段更新中任意一项失败时回滚整个事务。
- 前端读取失败时不展示默认徽标，编辑页默认按钮保持不可操作并展示现有错误反馈。
- 前端 mutation 失败时保留原缓存状态，并通过 Notification 告知用户。

读取失败属于明确错误状态，界面不会把失败响应解释为三个默认值均为空。

## 国际化与可访问性

新增文案同步维护 `en-US`、`zh-CN`、`zh-TW` 和 `ja-JP`，覆盖默认徽标、设置默认、取消默认以及资产不可用错误。

默认值按钮提供 tooltip、`aria-label`、键盘焦点样式和禁用状态。移动端继续使用现有顶栏图标按钮尺寸，列表徽标不得挤压资产名称和现有状态信息。

## 测试与验收

数据库迁移和持久化测试覆盖首次 upsert、已有记录更新、字段省略时保持原值、显式 null 清空、资产删除后外键置空、用户删除后级联删除。

应用层测试覆盖本人角色卡、本人预设、本人模型连接、他人公开资产、缺失资产、空请求、幂等更新和多字段原子回滚。HTTP 测试覆盖认证、GET 的未配置结果、PATCH 的完整响应、参数校验及错误信封。

前端测试覆盖三类编辑页的未选中和已选中图标、设置与取消操作、mutation 期间禁用、失败 Notification、非本人资产和新增页隐藏按钮、三类列表默认徽标，以及删除默认资产后的查询失效。OpenAPI 文档和 Web 类型随接口实现同步生成。

浏览器验收覆盖桌面与窄屏视口，检查顶栏操作顺序、tooltip、键盘焦点、徽标布局、长名称截断和删除后的状态同步。

## 范围外事项

- 聊天创建、聊天持久化和默认值的实际预选逻辑。
- Persona 选择器与对话角色选择器。
- 默认对话角色、默认世界书和其他聊天偏好。
- 他人公开角色卡或预设作为默认值。
- 集中的聊天偏好设置页面。
- 独立的默认模型或默认 API Key。
