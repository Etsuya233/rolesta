# Model Provider 界面与交互优化设计

## 背景

当前 Model Provider 编辑器将密钥选择与密钥 CRUD 集中在同一个弹窗中，可用模型也占用独立折叠分组。此次优化将密钥管理提升为独立页面，收紧编辑器中的密钥选择职责，并将模型加载与选择合并到“模型”分组。

本次保留现有 Model Provider API、数据库结构、页面栈、草稿会话和保存协议。

## 目标

- 避免浏览器密码管理器识别、保存或自动填充 Model Provider 密钥，同时保留默认遮挡和显示切换。
- 提供可从 Model Provider 首页和编辑器进入的独立密钥管理页。
- 使用紧凑、易区分的密钥列表完成创建、编辑、删除和排序。
- 将编辑器中的密钥入口简化为选择菜单，并提供前往密钥管理页的入口。
- 将“默认模型”和“可用模型”合并为一个使用 Select 的“模型”分组。
- 保持页面跳转前后的未保存草稿和表单状态。

## 页面结构与导航

Model Provider 页面栈新增 `api-keys` 页面类型。

Model Provider 首页顶栏在新增按钮左侧增加 `KeyRound` 图标按钮。按钮使用与新增按钮一致的 ghost icon 样式，不显示边框或文字，并通过 `aria-label` 表达“密钥管理”。点击后使用现有 `pushPage` 进入密钥管理页。

编辑器的密钥选择菜单底部提供“管理密钥”菜单项，同样使用 `pushPage` 进入密钥管理页。返回操作使用 `popPage`。从编辑器进入时，keep-alive 页面栈继续保留原编辑页及其草稿、折叠状态和表单内容。

新增组件职责如下：

- `ModelProviderApiKeyManagementPage`：提供顶栏、返回导航和页面布局。
- `ModelProviderApiKeyListPanel`：负责查询、排序、加载状态、错误状态和空状态。
- `ModelProviderApiKeyListItem`：呈现紧凑条目及编辑、删除操作。
- 密钥创建与编辑弹窗：负责表单输入和对应 mutation。
- 密钥删除确认弹窗：负责加载引用数量并执行删除。
- `ModelProviderCredentialField`：只处理手动密钥、关联密钥展示、解除关联和密钥选择。

现有 `ModelProviderApiKeyVaultDialog` 的 CRUD 职责迁移至管理页，选择职责迁移至 Dropdown Menu，原组件随之移除。

## 密钥管理页

顶栏标题为“密钥管理”，右侧使用 `Plus` 图标按钮创建密钥。内容区顶部放置现有样式的排序菜单，支持以下字段：

- 更新时间
- 创建时间
- 名称

每个字段均支持升序和降序，默认使用更新时间降序。

密钥条目采用紧凑单行布局，以细分隔线区分。左侧显示名称和更新时间，右侧显示编辑与删除图标按钮。图标按钮具有 tooltip 和独立可访问名称，点击按钮不会触发行级行为。

创建弹窗包含必填的名称和密钥字段。编辑弹窗包含名称和新密钥字段；新密钥留空时只更新名称。删除前先读取引用数量，确认内容明确显示受影响的 Model Provider 数量；引用数量加载失败时禁止确认并允许重试。

## 编辑器密钥交互

密钥字段右侧的 `KeyRound` 按钮打开 Dropdown Menu。菜单主体列出当前全部密钥，始终按名称 A-Z 排序，当前关联项显示选中标记。分隔线下方固定“管理密钥”菜单项，点击后关闭菜单并进入密钥管理页。

选中密钥后，字段显示关联的密钥名称并处于只读状态，点击文本区域不会解除关联或进入编辑。字段最右侧显示 `X` 图标按钮；点击后清除 `apiKeyId` 和 `apiKeyName`，切换到手动模式，并将手动密钥设置为空。

手动模式保留显示与隐藏按钮。输入元素使用普通文本语义，遮挡由界面样式控制，使浏览器密码管理器不将其识别为密码字段，也不触发保存或自动填充提示。

## 模型交互

删除独立的“可用模型”折叠分组。原“默认模型”分组改名为“模型”，其中仅保留一个 Select 字段。

模型分组展开且 Base URL 已设置时，根据 Provider 类型、Base URL、密钥模式和当前密钥信息加载远端模型列表。Select 在加载期间禁用；加载失败时显示错误与重试操作；返回空列表时显示明确空状态。

Select 只允许选择远端接口本次返回的模型，不提供任意文本输入，也不保留远端列表中不存在的历史值。选中结果写入现有 `defaultModelName` 字段，因此提交协议和持久化结构保持不变。

## 数据流与排序隔离

密钥 CRUD 继续使用现有 API。React Query 缓存保存服务端返回的原始密钥列表，不在缓存数组上执行原地排序。

密钥管理页使用 `useMemo` 根据本地排序字段和方向派生展示数组。时间字段按数值排序，名称使用当前语言环境比较。编辑器 Dropdown Menu 使用独立的 `useMemo` 派生名称 A-Z 数组。因此管理页的排序选择和方向不会改变选择菜单的顺序。

CRUD 成功后更新统一的密钥查询缓存，使管理页和选择菜单同步获得最新数据。新增和改名后刷新列表；删除后移除缓存条目，并使引用它的 Model Provider 查询失效。

删除密钥时，草稿会话容器同步清理所有仍挂载且引用该密钥的草稿：清除 `apiKeyId` 和 `apiKeyName`，切换为手动模式，并保持手动密钥为空。返回编辑页后不会出现已删除的密钥关联。

模型查询键继续包含 Provider 类型、Base URL、密钥模式、手动密钥和密钥 ID。连接信息变化后，模型 Select 使用对应查询结果。

## 错误处理

- 密钥列表加载失败时在页面内显示错误与重试操作。
- 创建或编辑失败时在弹窗内显示错误，并保留用户输入。
- 引用数量加载失败时禁止确认删除，并允许重试。
- 模型加载失败时保持 Select 禁用，并显示错误与重试操作。
- mutation 成功后才关闭对应弹窗，避免失败时丢失上下文。

## 国际化与可访问性

所有新增和调整文案同步维护 `en-US`、`zh-CN`、`zh-TW` 和 `ja-JP`。删除不再使用的“默认模型”“可用模型”和旧密钥弹窗文案键。

所有纯图标按钮提供 `aria-label`；编辑、删除等操作提供 tooltip。Dropdown Menu、Select 和弹窗沿用 shadcn 组件的键盘导航与焦点管理。紧凑列表仍保留适合触控的操作按钮尺寸，并对长名称进行截断。

## 测试与验收

组件测试覆盖：

- 首页密钥管理与新增图标按钮的顺序、样式和可访问名称。
- 密钥管理页默认按更新时间降序，以及三个字段和两个方向的排序。
- Dropdown Menu 始终按名称 A-Z 排序，不受管理页排序状态影响。
- 选择密钥后更新表单关联。
- 点击 `X` 后解除关联、清空密钥并恢复手动输入。
- 已关联密钥的文本区域无法编辑，点击文本区域不会解除关联。
- 从编辑器进入密钥管理页并返回后保留草稿。
- 删除密钥后清理所有仍挂载且引用它的草稿。
- 模型分组展开后加载列表，Select 选中值写入 `defaultModelName`。
- 模型加载中、失败、重试和空列表状态。
- 密钥输入保留遮挡和显示切换，同时不使用密码字段语义。

回归验证包括 Web typecheck、lint、相关组件测试和四语言资源覆盖测试。浏览器验收覆盖桌面与移动视口，检查顶栏、紧凑列表、弹窗、Dropdown Menu、Select、文字截断、触控面积、菜单边界以及返回后的草稿状态。

## 范围外事项

- 不修改密钥或 Model Provider 后端 API。
- 不修改数据库结构或提交协议。
- 不增加密钥搜索、分页、批量操作或密钥明文回显。
- 不允许输入远端模型列表以外的自定义模型名。
