# 预设关联模型连接设计

**状态：** 已确认
**日期：** 2026-07-14

## 目标

在预设创建和编辑表单中管理预设与模型连接的可选关联。关联只记录两项资产之间的关系，本次不接入聊天运行逻辑，也不扩展预设列表、SillyTavern 导入和导出格式。

## 领域规则

- 产品文案使用“模型连接”，代码继续沿用现有 `modelProviderId` 标识。
- 一个预设最多关联其所有者名下的一份模型连接，关联允许为空。
- 模型连接删除后，引用它的预设保留，关联自动置空。
- 预设模型连接关联只对预设所有者可见。其他用户读取公共预设时得到 `modelProviderId: null`。
- 创建或保存预设时，显式提交的非空关联必须可由当前用户访问。缺失、已删除和属于其他用户的连接统一视为资源不可用。

## 后端设计

### 请求与领域对象

`CreatePresetRequestDto` 增加可选且可空的 `modelProviderId`，`PresetEditableFieldsDto` 增加可选且可空的 `modelProviderId`，`UpdatePresetDocumentRequestDto` 增加必填且可空的 `modelProviderId`。预设创建、局部更新和整文档保存命令同步携带该字段。

`PresetEditableFields` 接受 `modelProviderId?: string | null`。局部更新省略字段时保留当前关联，提交 `null` 时解除关联，提交字符串时更新关联。整文档保存始终提交该字段。当前 `applyPresetEditableFields` 无条件清空关联的行为需要修正。导入预设继续创建为未关联状态。

常规预设聚合更新不写入模型连接关联列，防止条目或提示词保存把并发删除已经清空的关联重新写回。创建预设会随聚合首次保存关联；局部更新明确提交该字段或整文档保存时，才通过独立的关联写入操作更新该列。

### 所有权校验

创建、局部更新和整文档保存用例在收到非空 `modelProviderId` 时，通过预设应用层定义的 `PresetModelProviderAccess.acquireOwned(modelProviderId, ownerUserId)` 验证模型连接属于预设所有者，并保证它在当前 Unit of Work 完成前不会被并发删除。校验失败时抛出预设应用错误 `model-provider-unavailable`，HTTP 层映射为 `404 NOT_FOUND`。不存在与无权访问使用相同响应，避免暴露其他用户的资产信息。

`KyselyPresetModelProviderAccess` 只查询模型连接 ID 和所有者。Postgres 与 MySQL 在事务中对匹配行使用 `FOR UPDATE`，SQLite 在同一事务中完成读取和预设写入；并发删除或保存发生写冲突时，只允许其中一个事务提交。该适配器由 `PresetsModule` 直接注册，无需导出模型连接仓储，也不会读取 Base URL、凭据或默认模型等连接内容。

### 删除与一致性

`DeleteModelProviderUseCase` 使用现有 Unit of Work 包裹删除，并在成功删除后同步发布 `ModelProviderDeletedEvent`。预设上下文的监听器消费事件，通过 `PresetStore` 清空当前所有者预设中对应的 `model_provider_id`。事件监听与删除共享当前事务，任一监听器失败都会回滚删除及关联清理。

删除语句会等待保存事务持有的模型连接行锁；保存先提交时，删除事件随后清空新关联，删除先提交时，保存的关联校验失败。SQLite 的冲突事务无法同时提交。未授权删除影响行数为零，此时不发布事件。当前数据库列已经存在，本次不增加跨方言外键迁移。

### 公共预设响应

预设详情响应映射接收查看者 ID。查看者为所有者时返回实际 `modelProviderId`；其他用户读取公共预设时返回 `null`。领域对象和数据库记录保持原关联，脱敏只发生在 HTTP 响应边界。

## 前端设计

`PresetDocument` 和 `PresetEditorFormState` 纳入 `modelProviderId`，详情水合、脏状态比较、创建请求和整文档保存请求均包含该字段。

预设主编辑器使用现有模型连接分页接口加载当前用户的全部候选。前端 API 层聚合所有分页结果，保证普通 Select 不会遗漏后续页面。候选加载使用独立查询缓存，不改变模型连接管理页的查询状态。

基础信息区将只读的“未关联”徽标替换为 Select 和清除图标按钮。Select 选项以模型连接名称为主，并显示 Provider 类型和默认模型以区分同名项。清除按钮将 `modelProviderId` 设为 `null`，提供可本地化的可访问名称和悬停提示。保存中或候选加载中禁用控件；候选加载失败时显示表单内错误，当前预设草稿保持不变。

预设列表不展示关联信息。其他用户打开公共预设时收到空关联，也不会解析所有者的私有模型连接。

## 删除并发与错误行为

模型连接通过正常删除流程移除时，已有预设关联自动置空，不向用户报告关联错误。若编辑器持有过期草稿，并在连接删除后仍显式提交旧 ID，保存返回 `404 NOT_FOUND`；界面沿用现有保存错误通知，用户可以清除或重新选择。服务端不会静默改写这类明确提交。

## 测试范围

后端应用测试覆盖：

- 创建预设并关联自己的模型连接。
- 创建或更新时拒绝缺失及其他用户的模型连接。
- 局部更新省略字段时保留关联，提交 `null` 时解除关联。
- 整文档保存更新和清除关联。
- 删除模型连接后通过同步领域事件在同一事务内解除已有预设关联。
- 所有者读取真实关联，其他用户读取公共预设时获得空关联。

前端测试覆盖：

- 预设详情正确水合 `modelProviderId` 并参与脏状态比较。
- Select 修改关联，清除按钮将关联设为空。
- 创建请求和整文档保存请求携带关联字段。
- 候选分页聚合完整，加载状态和错误状态不丢失草稿。
