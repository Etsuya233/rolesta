# Rolesta

Rolesta 是一个面向角色扮演聊天的创作和对话平台，重点支持一对一角色扮演聊天、兼容资产管理和上下文调试。

平台优先兼容 SillyTavern 用户熟悉的角色卡、世界书和预设格式。用户可以把已有素材导入 Rolesta，在平台内继续整理、编辑、组合和聊天。导入时会尽量保留原始内容快照，方便排查差异、重新解释或后续导出。

## 产品定位

Rolesta 聚焦“可用、可解释、可持续整理”的角色扮演聊天工作台。

用户在平台中通过角色卡参与对话：角色卡既可以作为 AI 角色，也可以作为用户自己的聊天身份。当前核心形态是一对一聊天，后续可以在同一基础上扩展到更复杂的多人或群聊场景。

## 核心使用场景

### 导入和整理素材

用户可以把已有角色卡、世界书和预设导入 Rolesta，集中管理自己的角色、设定、提示词和模型配置。平台会把外部格式转换成内部对象，同时保留兼容信息，减少迁移过程中的内容损失。

### 创建聊天会话

用户选择自己的身份角色、对话角色、预设和模型档案后即可开始一对一聊天。聊天工作台围绕消息流展开，常用的角色信息、世界书、预设、模型和运行时信息放在靠近会话的位置，减少频繁跳转。

### 调整和复用上下文

用户可以通过角色卡、世界书和预设控制聊天上下文。世界书用于补充角色、地点、规则和背景设定；预设用于控制提示词编排和生成参数；变量目录帮助用户理解模板内容在生成前如何被替换。

### 调试生成结果

当回复不符合预期时，用户可以查看一次生成涉及的上下文摘要，例如命中的世界书条目、变量解析结果、请求消息摘要、历史截断情况和失败原因。这些信息用于解释回复为什么受到某些设定或提示词影响。

### 个人部署和扩展

Rolesta 面向个人部署场景设计。产品当前以稳定的聊天核心和资产管理为主，后续计划支持插件扩展聊天流程、工具面板、消息菜单和设置页。

## 当前产品边界

当前范围聚焦：

- 一对一角色扮演聊天。
- 角色卡、世界书和预设的兼容导入与管理。
- 模型档案配置。
- 基础上下文组装和调试。
- 桌面与窄屏下可完成核心聊天流程。

暂不作为当前完成范围：

- 完整复刻 SillyTavern 的所有行为。
- 实时多人群聊。
- 公开插件市场。
- 强隔离插件沙箱。
- 计费、组织权限和审计系统。
- 原生移动 App。

## 产品原则

- 聊天体验优先：用户进入产品后应能快速选择角色并开始对话。
- 兼容内容优先：已有角色卡、世界书和预设应尽量可导入、可编辑、可继续使用。
- 用户资产可控：用户应清楚知道自己拥有哪些角色、世界书、预设、模型档案和会话。
- 上下文可解释：用户应能查看一次生成受到哪些内容影响。
- 移动端可完成核心流程：发送消息、重新生成、编辑消息、切换预设和查看调试摘要等操作应能在窄屏完成。

## Development

Rolesta uses a TypeScript monorepo with a separated API and web app.

### Requirements

- Node.js 22 or newer.
- pnpm 9 or newer.

### Setup

```powershell
pnpm install --trust-lockfile
Copy-Item .env.example .env
pnpm db:migrate
```

### Run

```powershell
pnpm dev
```

Default local URLs:

- Web: http://127.0.0.1:5173
- API: http://127.0.0.1:3000
- API docs: http://127.0.0.1:3000/docs

### Verify

```powershell
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```
