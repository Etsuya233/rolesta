# Rolesta 产品功能说明

Rolesta 是一个面向角色扮演聊天的创作和对话平台。它的核心目标是让用户可以导入、整理和使用角色卡、世界书、预设等内容，并围绕这些内容进行稳定的一对一角色扮演聊天。

产品优先兼容 SillyTavern 用户熟悉的内容格式和使用习惯。用户可以把已有角色卡、世界书和预设带入 Rolesta，在平台内继续编辑、组织、聊天和调试。

## 产品定位

Rolesta 聚焦“可用的一对一角色扮演聊天核心”。一对一聊天被视为多人会话的一种简单形态，因此产品后续可以扩展到更复杂的群聊场景。

平台中的用户也通过角色卡参与聊天。也就是说，用户可以为自己选择一个角色身份参与对话，产品不额外设置独立的 User Persona 概念。

Rolesta 兼容外部格式，但会把导入内容转换成自己的产品对象。导入时保留原始内容快照，方便用户以后排查导入差异、重新解释内容或导出兼容文件。

## 目标用户

- 使用 SillyTavern 或类似工具进行角色扮演聊天的用户。
- 拥有角色卡、世界书、预设等素材，并希望集中管理这些素材的用户。
- 需要查看上下文、世界书命中、变量替换和生成过程摘要的进阶用户。
- 希望在个人部署环境中扩展聊天流程和界面能力的用户。

## 核心使用场景

### 初始设置与登录

用户第一次打开平台时，可以完成首个管理员账号设置。完成设置后，用户通过登录进入工作台。平台按用户隔离资产、会话和设置，避免不同用户的数据互相可见。

### 角色卡管理

用户可以管理自己的角色卡。角色卡既可以作为聊天中的 AI 角色，也可以作为用户自己的聊天身份。

角色卡功能包括：

- 创建新角色卡。
- 查看角色卡列表。
- 搜索和筛选角色卡。
- 编辑角色名称、描述、性格、场景、首条消息和示例对话。
- 编辑创作者、标签、版本、创作者备注、系统提示和后置提示等兼容字段。
- 添加和编辑备用开场白。
- 导入 Character Card JSON。
- 导入包含角色卡信息的 PNG。
- 导出 Character Card JSON。
- 查看导入时保留的兼容信息摘要。

角色卡导入后，用户可以继续编辑平台识别出的字段。未直接展示的兼容内容仍应被保留，保证后续导出或重新解释时尽量不丢失信息。

### 世界书管理

世界书用于保存角色、地点、背景设定、规则和其他可被聊天上下文引用的信息。用户可以创建多个世界书，并在其中维护条目。

世界书功能包括：

- 创建、查看、编辑和删除世界书。
- 创建、查看、编辑和删除世界书条目。
- 设置条目的关键词和辅助关键词。
- 设置条目是否启用。
- 调整条目优先级。
- 调整条目插入位置。
- 设置条目扫描深度和递归匹配选项。
- 重排条目的手动顺序。

聊天时，平台根据当前消息和近期上下文匹配世界书条目，并把命中的内容纳入生成上下文。

### 预设管理

预设用于控制聊天时的提示词编排、模型参数和部分行为提示。Rolesta 支持导入、编辑和导出与 SillyTavern 习惯接近的预设。

预设功能包括：

- 创建预设。
- 查看预设列表。
- 导入预设 JSON。
- 编辑预设名称。
- 设为默认预设。
- 编辑模型相关参数。
- 管理提示词条目。
- 启用或禁用提示词条目。
- 调整提示词条目顺序。
- 编辑单个提示词条目的名称、角色、内容和插入配置。
- 导出兼容 JSON。

预设导入后，平台展示当前支持的可编辑内容。暂未支持的内容需要保留在导入快照中，避免导出时丢失用户原有配置。

### 变量目录

平台提供变量目录，说明聊天上下文中支持的变量以及它们的含义。常见变量包括角色名、用户身份名、场景和每次生成中的随机值。

变量目录的目标是让用户理解自己的角色卡、预设和上下文模板在实际聊天时会如何被解释。

### 模型配置

用户可以配置用于聊天生成的模型档案。模型档案描述要使用的模型、连接信息和默认生成参数。

模型配置功能包括：

- 创建模型档案。
- 查看模型档案列表。
- 编辑模型档案。
- 启用或禁用模型档案。
- 设置默认模型档案。
- 删除未被使用的模型档案。
- 查看某个模型档案是否已配置密钥。

产品界面不应直接显示敏感密钥内容。

### 会话与聊天

用户可以创建会话，并选择参与会话的角色卡。核心体验是一对一聊天：一个用户身份角色和一个 AI 角色进行对话。

聊天功能包括：

- 创建会话。
- 查看会话列表。
- 打开会话详情。
- 修改会话标题。
- 归档会话。
- 选择会话使用的预设。
- 选择会话使用的模型档案。
- 查看分页消息。
- 发送用户消息。
- 接收流式生成中的助手消息。
- 编辑已有消息。
- 删除消息。
- 对指定消息重新生成回复。
- 查看失败消息的错误摘要。

聊天过程中，平台负责组装上下文。用户在界面中只需要发送消息、选择角色、选择预设和模型档案。

### 生成上下文与调试

Rolesta 面向进阶用户提供上下文调试能力，用来解释一次回复是如何生成的。

上下文调试功能包括：

- 查看一次生成所使用的请求消息摘要。
- 查看命中的世界书条目。
- 查看变量解析结果。
- 查看上下文片段和估算长度。
- 查看历史消息截断情况。
- 查看生成失败原因。

这个功能用于帮助用户理解为什么某次回复会受到某些角色设定、世界书条目或预设内容影响。

### 工作台体验

Rolesta 的主界面是一个紧凑的聊天工作台。产品重点是让常用操作靠近聊天区域，减少在多页面之间来回跳转。

工作台包含：

- 聊天视图。
- 资产视图。
- 设置视图。
- 会话导航。
- 角色、世界书、预设、模型和调试工具面板。
- 运行时信息面板。
- 明暗主题切换。

桌面端以多栏工作区为主。窄屏和移动端优先展示消息流和输入区，并通过抽屉打开导航、工具和运行时信息。核心聊天操作应在手机上可完成。

## 当前产品边界

当前聚焦一对一角色扮演聊天、兼容资产管理和基础上下文调试。

暂不作为当前完成范围的能力包括：

- 完整复刻 SillyTavern 的所有行为。
- 实时多人群聊界面。
- 公开插件市场。
- 强隔离的插件沙箱。
- 计费、组织权限和审计系统。
- 完整的 SillyTavern 导出能力。
- 高级视觉化 Prompt 编排。
- 原生移动 App。

## 插件系统规划

Rolesta 计划支持个人部署信任模型下的插件系统。插件可以扩展聊天流程、工具面板、消息菜单、设置页和其他工作台位置。

插件相关的产品目标包括：

- 安装插件。
- 启用和禁用插件。
- 配置插件。
- 查看插件声明的能力。
- 在聊天工具栏、消息菜单、侧边面板和设置页中展示插件功能。
- 允许插件提供自己的页面或命令。
- 记录插件运行错误。

当前插件系统属于规划能力。现有产品界面中可以保留插件入口或占位，但不应把它视为已经完整可用的功能。

## 产品原则

- 聊天体验优先。用户进入产品后应能快速选择角色并开始对话。
- 兼容内容优先。已有角色卡、世界书和预设应尽量可导入、可编辑、可继续使用。
- 用户资产可控。用户应清楚知道自己拥有哪些角色、世界书、预设、模型档案和会话。
- 上下文可解释。用户应能查看一次生成受到哪些内容影响。
- 移动端可完成核心流程。发送消息、重新生成、编辑消息、切换预设、查看角色信息和打开调试摘要等操作应能在窄屏完成。
- 导入内容尽量不丢失。平台未展示或暂未支持的兼容字段仍应被保留。

## Development

Rolesta uses a TypeScript monorepo with a separated API and web app.

### Requirements

- Node.js 22 or newer.
- pnpm 9 or newer.

### Setup

```powershell
pnpm install --trust-lockfile
Copy-Item .env.example .env
pnpm db:migrate
```

### Run

```powershell
pnpm dev
```

Default local URLs:

- Web: http://127.0.0.1:5173
- API: http://127.0.0.1:3000
- API docs: http://127.0.0.1:3000/docs

### Verify

```powershell
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```
